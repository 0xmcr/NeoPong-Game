# Contributing to AeroCut AI

Thank you for your interest in contributing to AeroCut AI! We welcome contributions from developers, designers, writers, and makers of all backgrounds.

---

## 🛠️ Code Conventions

### Python (Backend)
*   We follow the standard [PEP 8](https://peps.python.org/pep-0008/) style guidelines.
*   Use descriptive, snake_case names for variables and functions.
*   Document public modules and classes with standard docstrings.

### TypeScript / React (Frontend)
*   We use ESLint and Prettier for code formatting.
*   Write modular, reusable functional components with hooks.
*   Maintain semantic HTML elements and unique IDs for interactive nodes.

---

## 🚦 How to Contribute

### 1. Reporting Bugs
*   Search current issues to make sure the bug hasn't already been reported.
*   Create a new issue describing the steps to reproduce the bug, your environment (OS, Python/Node versions), and the expected versus actual behavior.

### 2. Suggesting Enhancements
*   Open an issue stating the feature you want to add, why it is valuable to the community, and suggestions on how it can be built.

### 3. Submitting Pull Requests
*   Fork the repository and create a branch from `main`.
*   Make your changes, ensure they are clean, and document any new API endpoints.
*   Run backend tests using `pytest` to make sure nothing is broken.
*   Rebuild the frontend using `npm run build` to confirm zero compilation errors.
*   Submit a Pull Request with a clear title and description of your changes.

---
We look forward to your contributions!
